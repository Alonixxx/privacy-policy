Summary & Core Philosophy

Local-First and Private

The Smart Home Control Hub is built with a privacy-first, local-first philosophy. We believe your home activity is personal. All configurations, activity tracking, location coordinates, and security footage are stored directly on your physical hardware, never in our own centralized cloud servers.

This Privacy Policy explains how data is gathered and handled when you run the Smart Home Control Hub, connect integration clients (Tuya, eWeLink, Wyze, Alexa), and use mobile locator applications (OwnTracks).

Information We Collect

In order to consolidate your smart devices and perform automations, the following categories of data are processed within your hub environment:

Local Storage Credentials & Tokens

API Access tokens, passwords, and client secrets (e.g., Tuya API credentials, eWeLink login details, Wyze Bridge URLs, and Alexa webhook secrets) are stored locally to talk to device clouds.

Local Storage Telemetry & Status

Status details (on/off, brightness, power output, presence status, blinds positions) are requested from connected devices to keep the control dashboard accurate in real-time.

Local Storage Live Camera Media

Snapshots and feed captures from security cameras (e.g. Wyze cams connected via bridge) are saved locally as secure media logs within your personal Lakehouse directories.

Local Storage Geolocation Data

To run geofenced routines (e.g., turning off heaters when leaving home), latitude, longitude, and distance telemetry from OwnTracks are received and stored on the hub.

How We Use Your Data

Your local data is used strictly to enable key hub features, including:

  • Device Control: Executing actions (like toggling a light or opening rollers) via direct cloud integrations.
  • Automation Engines: Monitoring geofence presence and sensor triggers to trigger custom smart routines.
  • Security Log Retention: Saving historical logs of device events, camera frames, and Alexa audio alerts in local analytical databases (DuckDB) for diagnostic and dashboard review.

Absolutely no data is collected, harvested, or transmitted by the developer of this software. All communication happens directly between your server and the corresponding smart device API provider.

Data Storage & Security

Your private data is secured locally through the following measures:

  1. Database Storage: Event logs, user presence, and config parameters are stored in a local DuckDB and SQLite instance on your host machine.
  2. API Credentials: Any credentials saved in configuration files (.env, credentials database) stay entirely within your local filesystem boundary. We mask credentials on UI pages to protect them from screen shoulder surfing.
  3. Network Protection: External webhook calls (such as from Amazon Alexa to update status) require a private secret webhook token. You control access to this endpoint using your own secure reverse proxy or Tailscale VPN configurations.

Third-Party Integrations

When using this hub, communication is established with third-party servers to command your appliances. These third-party services maintain their own separate privacy policies:

  • Tuya Smart Cloud API: Coordinates command routing for Tuya/SmartLife switches and sensors.
  • eWeLink (CoolKit) Cloud API: Connects to eWeLink/Sonoff switches.
  • Wyze Labs (Local Bridge): Decodes camera streams. By using a local bridge, video stream headers do not exit to third parties other than standard Wyze video handshakes.
  • Amazon Alexa: Enables voice commands and speech synthesis alerts via secure Alexa webhook tokens. Voice commands are processed in Amazon's clouds according to Amazon's Alexa Privacy Policy.

Your Choices & Data Rights

Because you fully control the hardware running the Smart Home Control Hub, you possess total sovereignty over your data:

  • Access & Export: You can download or view all activity log databases directly from your file system.
  • Deletion: You can wipe all history databases, media snapshots, and configured credentials at any time by deleting the local database file or clearing configs from the developer settings.
  • Opt-Out: Disabling individual clients or removing location tracker links instantly stops data processing for that service.

Contact Developer

If you have any questions, suggestions, or concerns regarding your privacy or data practices within this integration project, you can contact the integration developer directly on GitHub through the project repository issues page.
